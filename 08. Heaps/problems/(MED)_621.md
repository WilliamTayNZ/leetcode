# LeetCode 621 — Task Scheduler (Heap)

## Core idea

Cooldown `n` means: once you run a task, the **same task** can’t appear again until **at least `n` other slots** happen.

Instead of tracking per-task cooldown timers, we schedule in **blocks (cycles)** of length:

- `cycle_len = n + 1`

Why? In any window of length `n+1`, the same task **cannot appear twice**.  
So each cycle can contain **at most one occurrence** of any task.

---

## Data structure

- Count task frequencies.
- Use a **max-heap of frequencies** (identities don’t matter for the invariant; only counts do).

Heap meaning at the start of each cycle:

- these are tasks that are currently **eligible to be used in this cycle**.

---

## Cycle simulation

For each cycle (outer loop):

1. Try to schedule up to `n+1` tasks (inner loop).
2. For each scheduled task:
   - pop the largest freq
   - decrement it
   - **do NOT push it back immediately**
   - store it in `to_restore`
3. After the cycle ends:
   - push everything in `to_restore` back into the heap

**Why the inner loop enforces cooldown**

- Once a task is used in a cycle, it is removed from the heap and only reinserted **after** the cycle.
- Therefore, it **cannot be picked twice within the same `n+1` block**.
- That guarantees identical tasks are spaced by at least `n+1` slots → cooldown satisfied.

---

## Time accounting

Let `k = number of tasks actually run in this cycle`.

- If heap is **still non-empty after restore**:
  - we still have work left, so this cycle costs the **full `n+1` slots**
  - any unused slots were forced idles
- Else (no work left):
  - last cycle costs only `k` (no trailing idle needed)

So:

- `time += (n+1)` if remaining work exists
- otherwise `time += k`

---

## Small example

`A:3, B:3, C:2, n=2 (cycle=3)`

Schedule by cycles:

- Cycle 1: A B C
- Cycle 2: A B C
- Cycle 3: A B

No task repeats inside a cycle, so repeats are ≥ 3 apart.

---

## Key invariant (one-liner)

**“At most one occurrence per task per cycle of length `n+1`” ⇒ cooldown automatically holds.**
